# Newman 模块化开发计划总览

> 基于 Newman PRD v9.0 · 2026 · 内部文档

---

## 一、项目概述

本文档将 Newman PRD v9.0 拆分为 **14 个独立开发模块**，按 **4 个交付阶段**组织，预估总工期 **103 工作日**。每个模块有独立的 PRD 文档，可直接用于开发任务分配和验收。

---

## 二、阶段规划

| 阶段 | 模块 | 工日 | 核心交付 |
|------|------|------|----------|
| **Phase 1：可用内核** | M01–M06, M14 | 44 | Provider、Memory、Tools、Runtime、Sandbox、API/SSE、Config |
| **Phase 2：知识与记忆** | M07–M08 | 17 | RAG 全链路、统一错误恢复 |
| **Phase 3：扩展生态** | M09–M11 | 19 | Plugin/Skill 体系、MCP 接入、Scheduler |
| **Phase 4：产品化** | M12–M13 | 23 | 完整前端、企业 IM 接入 |

## 五、并行开发策略

### Phase 1 内部并行

- **第一批（并行）**：M14 Config、M01 Provider、M05 Sandbox — 无前置依赖
- **第二批（并行）**：M02 Memory（等 M01）、M03 Tools（等 M05）
- **第三批**：M04 Runtime — 汇聚 M01 + M02 + M03
- **第四批**：M06 API & SSE — 等 M04

### 跨阶段提前启动

- M07 RAG 仅依赖 M01，可在 Phase 1 后期提前启动解析子任务
- M12 Frontend 可在 Phase 2 后期启动基础框架搭建

## 七、风险提示

1. **M04 Runtime** 是最大风险点，依赖三个模块且逻辑复杂，建议先定义接口契约再并行开发
2. **M07 RAG** 工期最长（12天），建议拆分为"解析+索引"和"检索+引用"两个子任务
3. **M12 Frontend** 工期最长（15天），可在 Phase 2 后期开始基础框架搭建
4. 所有模块应在开发前先定义 **接口契约（Interface Contract）**，确保并行开发时不阻塞
